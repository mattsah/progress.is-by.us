Give Your Stylesheets a Clue

The inKLing Page Structure

A basic inKLing page has a linear top-level structure in which all major elements on a page are direct children of the body. Each of these elements should represent a semantic content segment. They should be one of the following:

  • header
  • article
  • section
  • footer

Purely structural div elements are not welcome. This means that you are either providing a page level header / footer, an article, or a section. Structural divs, such as those that would encapsulate a major piece of content vs. a secondary navigation or sidebar should be placed inside a section.

You should use the role="main" attribute and value to signify which section or article is the main piece of content for the page.

All children of body's content segments (thus, grandchildren of the body) which do not have a class of .billboard are constrained horizontally to 70% of the body width.

A .billboard, such as the parent of this paragraph is not constrained in the same way other grandchildren of the body are, however, all its children are. This enables the easy creation of full-width content islands at any point in the flow of the content segment.

Groups and Sets

The group and set class are pseudo-semantic classes which can be added to parent elements to change the default layout properties of their children and also to encapsulate (when necessary) floated elements. There are three "hints" which can be combined with either a .group or a .set:

Featurette

50% Left

A featurette provides a simple way to split content 50/50. Like all .group and .set elements, it has a 4rem gutter.

50% Right

In addition to the gutter, there is a 4rem margin placed on the top and the bottom of .group and .set elements in order to differentiate them from surrounding content. The only exception to this is when they immediately precede another heading.

Brochure

A .brochure.group works exactly the same as a .featurette.group, except that each child will be provided one-third of the content space to consume.

Left Side Content

On .group elements, the gutter itself is provided as a left margin to all elements. Clears are specifically placed on the hinted nth-child, and the left margin is removed from the first elements on new rows.

Middle Content

The width of child elements within a .group is determined by a calculation of d - m + (m / c) where d is the desired width expressed as a percentage, m is the desired gutter/margin, and c is the count of elements per row.

Right Side Content

For example, the width calc() for these brochure elements is as follows: 

33.3334% + 4rem - (4rem / 3)

Catalogue

Left Side Content

The .catalogue.group acts just like the aforementioned group hints, but expects 4 children per row for a desired width of 25%.

Middle Content

Catalogues are best used when you have a high number of items which need to present some introductory information, but maintain a terse structure.

Middle Content

Keep in mind that you don't need exactly 4 or a multiple thereof of child elements. It is possible to add 5 elements to a .catalogue.group and the fifth will simply remain floated left.

Right Side Content

Be advised, however, that if the content of the children is not roughly the same height, then the additional whitespace caused by elements which are horizontally aligned with one another can be disconcerting.

Container Groups

Container groups work precisely the same as regular groups, so you can hint them with .featurette, .brochure, etc, such that you might have a .container.featurette.group. The primary differenc is that in addition to gutter/margin, each child element is supplied a certain amount of padding.

Because of their nature, container groups are mostly designed for cases where one might apply a background color to the child elements to block them off and distinguish them further. Since it is often the case that content heights might differ, container groups will also ensure that the background on all children is filled.

Headings

All headings consume 5rem by default. Although their heights will differ, their top-margin is adjusted according to their height such that they do not break the 5rem limitation. The bottom margin is always 1rem.

Messaging

Messages are a simple way to provide information to your users. These can be used with everything from validation notices to simple informational helpers an coaching text.

Every message starts as a div.message containing one ore more child elements and an optional type hinter.

By default, a div.message will look like the first message in this list, however, you can apply the additional classes as follows:

  • notice - to convey acive information the user should take interest in.
  • success - to convey a successful action completed by the system.
  • warning - to convey information which someone needs to take not of.
  • error - to convey that an action has failed.

You can also apply the .message and message type hint classes to other elements, such as a form in order to create an alert of sorts.

About the only place that .message is not recommended, is directly on text parent such as being put on the p tag itself.
